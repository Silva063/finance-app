# Финансы — Руководство пользователя

Приложение состоит из двух независимых модулей:
- **Финучёт** — учёт приходов и расходов
- **Инвентаризация** — подсчёт наличных по купюрам и электронных кошельков

Переключение между модулями — кнопки **Финучёт / Инвентаризация** в верхней панели (на мобильных — нижняя навигация).

---

## 1. Финучёт

### 1.1 Добавить операцию

**Через кнопку:**
1. Нажмите **+ Операция** в боковом меню (или **+** на мобильном — плавающая кнопка в правом нижнем углу).
2. Заполните форму:
   - **Дата** — по умолчанию сегодня
   - **Тип** — Приход или Расход
   - **Способ** — Безналичный или Наличный
   - **Сумма** — всегда положительное число (тип определяет направление)
   - **Категория** — необязательно
   - **Комментарий** — необязательно
3. Нажмите **Сохранить**.

**Через быстрый ввод (без открытия формы):**

Поле внизу бокового меню (на мобильных — кнопка ⌨ в правом углу экрана).

Формат строки:
```
[+/-][сумма] [нал|безнал] [комментарий]
```

Примеры:
| Ввод | Результат |
|---|---|
| `150 нал продукты` | Приход 150 руб, наличный, «продукты» |
| `+5000 безнал зарплата` | Приход 5 000 руб, безналичный, «зарплата» |
| `-80 нал` | Расход 80 руб, наличный |
| `200` | Приход 200 руб, безналичный |

Знак `+` или без знака — приход, `-` — расход. Слово `нал` (или `наличный`) — наличный способ, иначе безналичный.

### 1.2 Редактировать / удалить операцию

Нажмите кнопку **✎** справа в строке таблицы — откроется та же форма. Кнопка **Удалить** появляется только при редактировании.

### 1.3 Детализация позиций

В форме операции есть раздел **Детализация позиций** (раскрывается стрелкой). Два режима:

- **Детализация ↓** — общая сумма известна, расписываете из чего состоит. Показывает остаток/превышение от суммы.
- **Сборка ↑** — сумма неизвестна, складывается автоматически из позиций.

Добавьте позиции кнопкой **+ Позиция** (название, количество, цена). В таблице операций появится кнопка **▸ N** — нажмите, чтобы развернуть список позиций прямо в таблице.

### 1.4 Шаблоны операций

Шаблоны позволяют сохранять часто повторяющиеся операции и заполнять форму в один клик.

**Сохранить шаблон:**
1. Откройте форму операции и заполните нужные поля (тип, способ, сумма, категория, комментарий).
2. Нажмите кнопку **💾 Сохранить шаблон** в верхней части формы.
3. Введите название шаблона и нажмите **OK**.

**Применить шаблон:**
- Кнопка **◩ Шаблоны** в боковом меню — открывает список всех шаблонов.
- Кнопка **◩ Из шаблона** в верхней части формы операции — позволяет выбрать шаблон не закрывая форму.
- Нажмите **Применить** рядом с нужным шаблоном — поля формы заполнятся автоматически, дата установится на сегодня.

**Удалить шаблон:**
Нажмите **✕** рядом с шаблоном в списке и подтвердите удаление.

### 1.5 Разделы бокового меню

| Раздел | Содержимое |
|---|---|
| **Все операции** | Полный список с фильтрами и диаграммой по месяцам |
| **Безналичные** | Только безналичные + нарастающий итог (сверка с е-кошельком) |
| **Наличные** | Только наличные + нарастающий итог (сверка с кошельком) |
| **По месяцам** | Сворачиваемые блоки по месяцам, внутри — по дням |
| **Текущий месяц** | Таблица с нарастающими итогами по нал/безнал/общий |
| **Категории** | Диаграмма и таблица расходов/доходов по категориям |

### 1.6 Фильтры

На страницах «Все операции», «Безналичные», «Наличные» доступны фильтры:
- Поиск по тексту (комментарий, название позиций)
- Тип операции, способ оплаты, категория
- Год и месяц

Кнопка **↺ Сброс** снимает все фильтры.

### 1.7 Категории

**Страница «Категории»** показывает диаграмму (кольцо / круг / столбцы / полярная) и подробную таблицу. Фильтрация по году, месяцу, типу.

**Таблица содержит:**
- Сумму и долю (%) каждой категории
- Среднее значение в месяц (за выбранный период)
- **Тренд** — сравнение с предыдущим месяцем: зелёная стрелка ▲ — рост, красная ▼ — снижение (отображается только при фильтре по конкретному месяцу)
- **Строку «Итого»** в нижней части таблицы

**Лимиты расходов** — для каждой категории можно задать максимальную сумму расходов в месяц (подробнее — в разделе 1.8).

**Развернуть операции категории:** нажмите на строку категории в таблице — появятся последние 5 операций прямо под ней. Ссылка «ещё N →» переходит к полному отфильтрованному списку.

**Управление категориями** — кнопка **⚙ Категории** на этой странице:
- Переименовать, сменить цвет и иконку
- Задать лимит расходов в месяц
- Добавить новую категорию
- Удалить (операции с удалённой категорией остаются, категория помечается как «—»)

**Эмодзи категорий** — чекбокс «Эмодзи категорий» в том же окне включает отображение иконки перед названием (например, 🚗 Транспорт). По умолчанию выключено. Иконку каждой категории можно сменить: нажмите кнопку редактирования иконки рядом с нужной категорией и введите любой эмодзи. Новые категории создаются с иконкой 📌.

### 1.8 Лимиты категорий

Лимит — максимальная сумма расходов по категории за один месяц. Задаётся в настройке категорий (**⚙ Категории**) в поле «лимит» рядом с названием каждой категории. Оставьте поле пустым, чтобы убрать лимит.

**Как отображается прогресс (на странице «Категории»):**
- Тонкая полоса под названием категории в таблице — визуальный индикатор расходов относительно лимита
- **Зелёный** — до 80% лимита
- **Жёлтый (!)** — от 80% до 100% (близко к лимиту)
- **Красный (✕)** — лимит превышен

**Уведомления при сохранении операции:**
- Если расход приближается к лимиту (≥ 80%) — появляется предупреждение с текущей суммой и лимитом.
- Если лимит превышен — показывается предупреждение с суммой превышения.

### 1.9 Экспорт и импорт

Кнопка **↓ Экспорт** в боковом меню:
- **JSON** — полная резервная копия, подходит для импорта обратно
- **Excel (.xlsx)** — таблица операций для анализа в Excel или Google Sheets

Кнопка **↑ Импорт** — загрузите файл `.json` (полный импорт, заменяет текущие данные) или `.xlsx` (добавляет операции из таблицы).

При импорте XLSX ожидаемые столбцы: Дата, Тип (Приход/Расход), Способ, Сумма, Комментарий.

## 2. Инвентаризация

Модуль для подсчёта фактических наличных денег по купюрам и остатков на электронных кошельках. Поддерживает несколько валют и пересчёт в базовую валюту (по умолчанию — Рубль ПМР).

### 2.1 Первая настройка — блоки

Перед первым использованием создайте блоки через **⚙ Блоки** в боковом меню.

Блок — это отдельная «пачка» денег: кошелёк, стопка купюр, карта.

**Создать блок:**
1. Нажмите **+ Новый блок**.
2. Введите название (например: «Рубли ПМР», «Доллары», «Сбербанк»).
3. Выберите тип:
   - **Наличный (купюры)** — вводите количество каждого номинала по отдельности
   - **Электронный (одно поле)** — вводится одна сумма
4. Укажите **Валюту конвертации** — в какой валюте хранятся деньги в этом блоке.
5. **Учитывать в итоге ПМР** — если «Да», блок включается в основную итоговую сумму. Если «Нет» — блок считается, но в итог не идёт.
6. Для наличного типа введите **Номиналы** через запятую (например: `500, 200, 100, 50, 25, 10, 5, 1`).

Порядок блоков можно менять перетаскиванием за значок **⠿** в списке настройки блоков.

### 2.2 Курсы валют

Кнопка **₽ Курсы** в боковом меню. Здесь настраиваются курсовые пары (откуда → куда).

Каждая пара имеет два курса:
- **Покупка** — вы продаёте исходную валюту банку и получаете целевую
- **Продажа** — вы покупаете исходную валюту у банка, отдаёте целевую

При подсчёте инвентаризации используется курс **покупки** — сколько базовых рублей дадут за вашу иностранную валюту.

Для добавления новой валюты — кнопка **+ Добавить валюту** в настройке блоков (⚙ Блоки). Конвертация работает через промежуточные валюты: если прямого курса нет, алгоритм найдёт цепочку (например UAH → RUP через USD).

Кнопкой **Сменить базовую** в модале курсов можно выбрать другую базовую валюту.

### 2.3 Создать инвентаризацию

1. Нажмите **+ Новая дата** в боковом меню.
2. Выберите дату (по умолчанию сегодня) и при необходимости добавьте заметку.
3. Нажмите **Создать**.

В открывшейся форме введите:
- Для наличных блоков — количество купюр каждого номинала
- Для электронных блоков — сумму на счёте/кошельке

Итоговые суммы в базовой валюте пересчитываются автоматически.

### 2.4 Редактировать инвентаризацию

Нажмите на нужную дату в боковом меню — откроется та же форма заполнения. Кнопка **Удалить** удаляет запись целиком.

### 2.5 Главная (дашборд)

Кнопка **Главная** в боковом меню — обзор всех инвентаризаций.

**Два графика:**
- **ПМР и общий итог** — динамика общей суммы и суммы ПМР
- **Изменение ПМР** — разница между соседними датами (зелёный — рост, красный — снижение)

Кнопки диапазона над каждым графиком: **Все**, **12**, **6**, **3** (последние N месяцев). Кнопка **⚙** открывает выбор конкретных точек на графике.

Ниже графиков — таблица со всеми записями: дата, итог ПМР, общий итог, изменение.

### 2.6 Сравнить две инвентаризации

Кнопка **⇄ Сравнить** в боковом меню. Выберите две даты — получите постатейное сравнение по каждому блоку с разностью.

### 2.7 Цели

Кнопка **◎ Цели** в боковом меню. Создавайте финансовые цели с целевой суммой (в любой валюте). Прогресс считается автоматически по последней инвентаризации.

## 3. Google Drive — синхронизация

Приложение умеет сохранять данные в облако Google Drive и синхронизироваться между устройствами.

### 3.1 Подключить Drive

Нажмите на индикатор **Drive** в правом верхнем углу (или **☁ Drive** в нижней навигации на мобильном).

В открывшемся окне нажмите **Войти через Google** и выберите аккаунт. Токен кешируется на 50 минут — при следующем открытии приложения авторизация происходит автоматически (в фоне).

### 3.2 Ручная синхронизация

- Кнопка **⟳ Синхронизировать** в боковом меню Финучёта — синхронизирует только операции
- Кнопка **⟳ Синхронизировать** в боковом меню Инвентаризации — синхронизирует только инвентаризацию
- Кнопка **⟳ Синхронизировать всё** в окне Drive — синхронизирует оба модуля сразу

**Автосохранение:** каждое изменение данных автоматически отправляет обновление на Drive (с задержкой 2,5 сек).

### 3.3 Как работает слияние

При синхронизации данные из Drive и локальные данные объединяются умно:
- **Операции** сравниваются по ID — при конфликте побеждает версия с более поздней датой изменения
- **Настройки** (категории, блоки, курсы) берутся целиком с той стороны, которая изменялась позже

Удалённые операции не стираются навсегда — они помечаются флагом (мягкое удаление) и не отображаются, но сохраняются для корректного слияния с другими устройствами.

### 3.4 Папка на Drive

Приложение создаёт папку **FinanceApp** в корне Drive. Внутри:
- `finOps2026.json` — данные Финучёта
- `finData.json` — данные Инвентаризации
- `snapshots/` — снапшоты (если сохраняли с Drive)

## 4. Снапшоты — точки сохранения

Снапшоты позволяют зафиксировать полное состояние приложения и вернуться к нему в любой момент.

Кнопка **◈ Архив снапшотов** есть в боковом меню обоих модулей.

### Создать снапшот

1. Введите название (необязательно — если не вводить, сгенерируется автоматически с датой/временем).
2. Нажмите **💾 Локально** или **☁ + Drive** (второй вариант требует подключённого Drive).

Лимит локальных снапшотов — 50 штук. При превышении нужно удалить старые.

### Восстановить снапшот

Нажмите **↺ Загрузить** рядом с нужным снапшотом. Текущее состояние будет заменено данными из снапшота.

### Загрузить / скачать

- Значок **☁↑** — загрузить локальный снапшот на Drive
- Значок **☁↓** — скачать снапшот с Drive в локальное хранилище

Кнопка **☁ Список с Drive** — сканирует папку `FinanceApp/snapshots/` на Drive и добавляет найденные снапшоты в список.

## 5. Оформление

Кнопка **◑** в правом верхнем углу открывает настройки темы.

### Пресеты

Готовые цветовые схемы — нажмите на любой, тема применится мгновенно:

| Пресет | Описание |
|---|---|
| Тёмная | По умолчанию, тёмный фон + фиолетовый акцент |
| Светлая | Белый фон, синий акцент |
| Полночь | Очень тёмная, фиолетово-зелёная палитра |
| Сланец | Тёмно-синий фон, голубой акцент |
| Мокко | Коричневые тона, тёплый акцент |

### Пользовательская тема

**6 основных пикеров** — задайте главные цвета, остальные вычисляются автоматически:

| Пикер | Что управляет |
|---|---|
| **Фон** | Основной фон страницы |
| **Панели** | Сайдбар, топбар, фон карточек |
| **Акцент** | Кнопки, активные пункты меню, индикаторы |
| **Текст** | Основной цвет текста |
| **Доход** | Суммы доходов, позитивные значения |
| **Расход** | Суммы расходов, негативные значения |

Из этих шести приложение **автоматически выводит** промежуточные фоны (bg3, bg4), цвета линий/разделителей, светлый вариант акцента, приглушённые оттенки текста и фоновые подсветки семантических цветов.

Живой пример справа обновляется сразу при смене любого пикера. Нажмите **Применить** — тема сохранится и применится ко всему интерфейсу.

### Тонкая настройка

Раскройте секцию **▶ Тонкая настройка** под пикерами — там доступны все 15 CSS-переменных по группам:

- **Фоны** — `--bg`, `--bg2`, `--bg3`, `--bg4`
- **Линии** — `--line`, `--line2`
- **Акценты** — `--acc`, `--acc2` (светлый вариант)
- **Текст** — `--text`, `--muted`, `--muted2`
- **Семантика** — `--green`, `--red`, `--amber`, `--blue`

Изменения здесь применяются **сразу**, без кнопки «Применить», и сохраняются мгновенно.

### Экспорт и импорт

Кнопки в нижней части модального окна позволяют сохранить текущую тему в JSON-файл и загрузить её обратно. Удобно для переноса между устройствами или резервной копии.

Кнопка **Сброс** возвращает тему «Тёмная (по умолчанию)».

## 7. Хранение данных

Все данные хранятся в браузере (`localStorage`):
- `finOps2026` — операции Финучёта
- `finData` — данные Инвентаризации
- `finSnapshots` + `finSnapshots_data_*` — снапшоты

Данные не исчезнут при закрытии браузера, но удалятся если очистить данные сайта. Для надёжности используйте синхронизацию с Drive или регулярно делайте экспорт JSON.
